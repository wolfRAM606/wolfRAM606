# Sriram here — welcome to my corner of the internet

I'm a Computer Science student specializing in Business System at VIT Vellore who enjoys building and learning about software that solves real problems—not just because it's interesting to code, but because it's genuinely useful, no matter how tiny a problem it feels.

What keeps me hooked isn't a particular language or framework; it's understanding how systems fit together. Whether that's a web application communicating with a remote Linux server, an AI model learning to recognize Carnatic ragas, or a tool that makes everyday student life a little easier, I like figuring out what's happening beneath the surface and building something reliable on top of it.

Most of my projects start with a simple question: *"Could this be done better?"* From there, I end up exploring new technologies, reading documentation I probably shouldn't be reading at 2 a.m., and slowly turning an idea into something people can actually use.

I'm especially interested in applying AI/ML to real-world problems, along with backend engineering, systems programming, Linux-based systems, and building reliable infrastructure that supports practical, scalable applications.

### What I've been working on

**[DRDL Remote HPC Job Execution System](https://github.com/wolfRAM606/drdl-remote-hpc-job-execution-prototype)** — A web-based system for submitting and monitoring computational jobs on remote Linux servers over SSH. Developed as a public prototype inspired by work carried out during my internship at DRDL, DRDO. Features PBS/TORQUE-compatible scheduling, real-time monitoring, and intelligent error diagnostics. Python, Streamlit, Paramiko.

**[Raga Lens](https://github.com/wolfRAM606/raga-lens)** — AI-powered Carnatic raga detection using a DNN/CNN ensemble trained on 96 ragas. FastAPI backend, React frontend, TensorFlow models. Achieved ~70% accuracy on DNN with 96-dimensional MFCC+chroma+spectral features.

**[Campus Homie](https://github.com/wolfRAM606/Campus-Homie)** — A lightweight campus support platform built with React and Firebase. Students and faculty can post help requests, browse by category, and connect with each other. Simple idea, clean execution.

**[Recipeek](https://github.com/wolfRAM606/Recipeek---Usability)** — A mobile-first recipe app UI prototype built for usability testing. Six interactive screens covering recipe discovery, meal planning, and inventory management. Pure HTML/CSS/JS.

## How I like to build software

I enjoy building software that's meant to be maintained—not just demonstrated.

I care about writing code that's clean enough for someone else to read six months later and still understand what's going on. I think good software is as much about communication as it is about computation — clear naming, honest documentation, and a structure that makes sense without needing a guide. I'd rather spend extra time designing something clean than spend twice as long debugging unnecessary complexity later.

For me, good engineering isn't about using the newest technology—it's about solving the right problem in a way that's dependable, understandable, and easy for someone else to build on.
